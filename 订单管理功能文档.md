# 订单管理功能文档

## 功能概述

本系统实现了完整的用户预约订单管理功能，包括订单查询、修改、取消和通知机制。

## 二、功能详细说明

### 1. 订单查询功能

**页面位置**：`pages/orders/orders`

**功能特性**：
- 查看用户所有预约订单
- 支持状态筛选（全部/待确认/已确认/已完成/已取消）
- 下拉刷新功能
- 订单卡片展示关键信息

**展示信息**：
- 订单状态标签
- 预约日期和时间
- 拍摄地点
- 角色名称
- 订单ID

**数据存储**：
- 存储键：`photo_orders`
- 数据格式：数组

---

### 2. 订单详情功能

**页面位置**：`pages/order-detail/order-detail`

**展示信息**：
- 订单状态和ID
- 拍摄信息（日期、时间、时长、地点、人数）
- 角色和道具信息
- 参考图片预览
- 风格偏好（拍摄偏好、色调、特别说明）

---

### 3. 订单修改功能

**触发条件**：仅订单状态为 `pending`（待确认）时可修改

**可修改内容**：
- 预约日期
- 地点补充说明

**修改流程**：
1. 点击「修改预约」按钮
2. 弹出修改模态框
3. 选择新日期或输入补充说明
4. 点击「保存」
5. 系统更新订单并发送通知

**权限验证**：
```javascript
const canEdit = order.status === 'pending'
```

---

### 4. 订单取消功能

**触发条件**：订单状态为 `pending` 或 `confirmed` 时可取消

**取消流程**：
1. 点击「取消预约」按钮
2. 弹出确认对话框
3. 用户确认取消
4. 系统更新订单状态为 `cancelled`
5. 记录取消时间
6. 发送取消通知

**状态变更**：
- `pending` → `cancelled`
- `confirmed` → `cancelled`

**权限验证**：
```javascript
const canCancel = order.status === 'pending' || order.status === 'confirmed'
```

---

### 5. 通知机制

**通知类型**：

| 类型 | 触发条件 | 说明 |
|------|---------|------|
| `order_cancelled` | 用户取消订单 | 订单取消通知 |
| `order_updated` | 用户修改订单 | 订单修改通知 |

**通知存储**：
- 存储键：`photo_notifications`
- 数据结构：
```javascript
{
  _id: 'notif_xxx',
  type: 'order_cancelled',
  data: { orderId: 'xxx', message: '用户取消了预约' },
  read: false,
  createdAt: '2024-04-15T10:30:00.000Z'
}
```

**通知使用**：
- 通知存储在本地，可后续扩展为系统内消息推送
- 可在后台管理页面查看和管理通知

---

### 6. 数据校验和安全

#### 权限控制

| 操作 | 允许的状态 |
|------|-----------|
| 查看订单 | 所有状态 |
| 修改订单 | 仅 `pending` |
| 取消订单 | `pending` 或 `confirmed` |

#### 数据一致性

- 所有操作更新 `updatedAt` 时间戳
- 取消操作记录 `cancelledAt` 时间戳
- 使用事务性更新（本地存储直接覆盖）

## 三、数据结构

### 订单数据结构

```javascript
{
  _id: 'order_xxx',                    // 订单ID
  status: 'pending',                   // 状态：pending/confirmed/completed/cancelled
  preferredDate: '2024-04-15',        // 预约日期
  preferredTime: '10:00',              // 预约时间
  duration: 2,                         // 时长（小时）
  location: {
    name: '颐和园',                     // 地点名称
    address: '北京市海淀区',            // 详细地址
    latitude: 39.999,                  // 纬度
    longitude: 116.276                // 经度
  },
  locationNotes: '从东门进',           // 地点补充说明
  peopleCount: 1,                      // 拍摄人数
  props: {
    characterName: '芙莉莲',            // 角色名
    referenceImages: ['temp://xxx'],   // 参考图片
    propItems: ['prop-id-1'],          // 选择的道具
    notes: '需要魔法杖'                // 道具备注
  },
  stylePreferences: {
    shootPreference: 'closeup',        // 拍摄偏好
    colorR: 128,                       // 红色值
    colorG: 128,                       // 绿色值
    colorB: 128,                       // 蓝色值
    specialRequests: '想要清冷风格'    // 特别说明
  },
  createdAt: '2024-04-15T10:30:00.000Z',    // 创建时间
  updatedAt: '2024-04-15T10:30:00.000Z',    // 更新时间
  cancelledAt: '2024-04-15T11:00:00.000Z'   // 取消时间（可选）
}
```

## 四、使用流程

### 订单查询流程

1. 用户进入「订单」页面
2. 系统从本地存储读取订单列表
3. 根据选择的状态筛选订单
4. 展示订单卡片列表

### 查看订单详情流程

1. 用户在订单列表点击某个订单
2. 跳转到订单详情页面
3. 系统加载完整订单信息
4. 展示所有订单详情

### 修改订单流程

1. 在订单详情页点击「修改预约」
2. 弹出修改模态框
3. 用户修改日期或补充说明
4. 点击「保存」
5. 系统验证权限
6. 更新订单数据
7. 发送通知
8. 刷新页面显示

### 取消订单流程

1. 在订单详情页点击「取消预约」
2. 弹出确认对话框
3. 用户确认取消
4. 系统验证权限
5. 更新订单状态为 `cancelled`
6. 记录取消时间
7. 发送通知
8. 刷新页面显示
