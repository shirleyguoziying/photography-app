# 系统修复说明文档

## 一、问题总结与修复方案

### 问题1：模拟器启动失败
**原因**：多个页面依赖云开发（`wx.cloud`）和外部服务模块，在没有配置云开发环境时会导致崩溃。

**修复**：
- 简化 `app.js`，移除云开发初始化和外部模块依赖
- 简化各页面，移除对外部服务的依赖
- 使用本地存储（`wx.setStorageSync` / `wx.getStorageSync`）替代云数据库

---

### 问题2：拍摄人数选择加减号按钮无法使用
**原因**：缺少对应的事件处理函数。

**修复**：
- 在 `booking.js` 中添加 `incPeople()` 函数（增加人数）
- 在 `booking.js` 中添加 `decPeople()` 函数（减少人数）
- 人数范围限制：1-10人

```javascript
incPeople() {
  const count = this.data.formData.peopleCount || 1
  if (count < 10) {
    this.setData({ 'formData.peopleCount': count + 1 })
  }
},

decPeople() {
  const count = this.data.formData.peopleCount || 1
  if (count > 1) {
    this.setData({ 'formData.peopleCount': count - 1 })
  }
}
```

---

### 问题3：拍摄人物图片无法上传
**原因**：缺少图片选择和处理逻辑。

**修复**：
- 在 `booking.js` 中添加 `chooseReferenceImages()` 函数
- 使用 `wx.chooseImage()` API 选择图片
- 限制最多上传9张图片
- 在 `booking.js` 中添加 `removeReferenceImage()` 函数删除已选图片

```javascript
chooseReferenceImages() {
  const currentCount = this.data.formData.props.referenceImages.length
  const maxCount = 9 - currentCount
  
  wx.chooseImage({
    count: maxCount,
    sizeType: ['compressed'],
    sourceType: ['album', 'camera'],
    success: (res) => {
      const newImages = res.tempFilePaths
      const allImages = this.data.formData.props.referenceImages.concat(newImages)
      this.setData({
        'formData.props.referenceImages': allImages
      })
    }
  })
}
```

### 问题4：拍摄偏好功能显示内容
**原因**：缺少数据加载和配置。

**修复**：
- 从 `config/constants.js` 导入 `SHOOT_PREFERENCES` 配置
- 在页面 data 中设置 `shootPreferences` 数据
- 添加 `onShootPreferenceSelect()` 事件处理函数
- 配置完整的偏好选项：大头为主、半身为主、大景为主、氛围感爱好者、混合模式

```javascript
const { DURATION_OPTIONS, SHOOT_PREFERENCES } = require('../../config/constants')

Page({
  data: {
    shootPreferences: SHOOT_PREFERENCES,
    // ...
  },
  
  onShootPreferenceSelect(e) {
    const preference = e.currentTarget.dataset.value
    this.setData({
      'formData.stylePreferences.shootPreference': preference
    })
  }
})
```

### 问题5：RGB色环无法改变颜色
**原因**：缺少颜色更新逻辑和事件处理。

**修复**：
- 添加 `_updateColorHex()` 函数实时计算HEX值
- 添加 `onColorRChange()`、`onColorGChange()`、`onColorBChange()` 事件处理
- RGB值变化时自动更新HEX显示
- 使用 `bindchanging` 事件实时响应滑块变化

```javascript
_updateColorHex() {
  const { colorR, colorG, colorB } = this.data.formData.stylePreferences
  const toHex = n => Number(n || 0).toString(16).padStart(2, '0')
  const hex = '#' + toHex(colorR) + toHex(colorG) + toHex(colorB)
  this.setData({ colorHex: hex })
},

onColorRChange(e) {
  this.setData({
    'formData.stylePreferences.colorR': e.detail.value
  })
  this._updateColorHex()
}
```

## 六、常见问题

### Q: 为什么不用云开发？
A: 为了快速修复问题，先用本地存储。如需云开发，请参考 `云开发使用指南.md` 配置。

### Q: 图片会永久保存吗？
A: 当前方案使用临时文件路径，重启小程序后会失效。如需持久保存，需要配置云存储。

### Q: 数据会丢失吗？
A: 本地存储在用户清除小程序缓存或卸载时会丢失。如需持久化，建议配置云开发。

### Q: 如何切换到云开发？
A: 参考 `云开发使用指南.md` 配置环境后，修改相关页面使用 `wx.cloud.database()` API。
